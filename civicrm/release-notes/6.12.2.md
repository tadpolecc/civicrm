## <a name="bugs"></a>Bugs resolved

* **_CiviContribute_: Currency info is not sent to some "Payment Processors" (e.g. PayPal Standard) ([dev/core#6396](https://lab.civicrm.org/dev/core/-/issues/6396): [#35188](https://github.com/civicrm/civicrm-core/pull/35188))**
* **_CiviMember_: Cannot edit new "Membership Types" ([dev/core#6389](https://lab.civicrm.org/dev/core/-/issues/6389): [#35165](https://github.com/civicrm/civicrm-core/pull/35165))**
* **_Composer_: Update "phpseclib" ([#35148](https://github.com/civicrm/civicrm-core/pull/35148))**

  phpseclib v3.0.50 is nominally a security update, but the issue does not affect normal CiviCRM configurations. Never-the-less, auditing tools may make noise about it. This update suppresses the noise.

* **_Contact Search_: Preview dialog ("Overlay") has malformed email link ([dev/core#6393](https://lab.civicrm.org/dev/core/-/issues/6393): [#35182](https://github.com/civicrm/civicrm-core/pull/35182), [#35192](https://github.com/civicrm/civicrm-core/pull/35192))**
* **_From Email Address_: Double-encoding in dropdown lists ([#35153](https://github.com/civicrm/civicrm-core/pull/35153))**
* **_Riverlea_: Description is missing from settings page ([#35150](https://github.com/civicrm/civicrm-core/pull/35150))**
* **_Upgrader_: Error about "hook_permission" when removing "minifier" ([dev/core#6394](https://lab.civicrm.org/dev/core/-/issues/6394): [#35160](https://github.com/civicrm/civicrm-core/pull/35160))**
